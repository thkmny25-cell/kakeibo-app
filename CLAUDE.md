# kakeibo-app

## プロジェクト概要

家計簿アプリ（kakeibo-app）の開発リポジトリ。

## Git 運用ルール

**コードを変更するたびに、必ず GitHub にプッシュすること。**

具体的な手順：

1. 変更をステージングする
   ```
   git add <変更ファイル>
   ```
2. コミットメッセージは変更内容を端的に記述する
   ```
   git commit -m "変更内容の説明"
   ```
3. GitHub にプッシュする
   ```
   git push origin <ブランチ名>
   ```

- `main` ブランチへの直接プッシュは避け、機能ブランチを作成して PR を経由する
- コミットメッセージは日本語でも英語でも可。ただし内容が明確に伝わるように書く
- 破壊的な操作（`git push --force`、`git reset --hard` 等）は必ずユーザーに確認してから実行する

## ファイルパスの記載

ファイルやコードに言及するときは、必ずファイルパスを記載すること。

例：ファイル：src/components/ExpenseForm.tsx

## 動作テスト

Web 画面を作成・更新したら、必ず PlaywrightMCP を使って動作テストを行うこと。

## コーディング方針

- コメントは原則書かない。WHY が非自明な場合のみ一行で記述する
- エラーハンドリングはシステム境界（ユーザー入力・外部 API）のみに限定する
- 不要な抽象化・将来の要件への備えは行わない
- セキュリティ上の脆弱性（XSS、SQL インジェクション等）を導入しないよう注意する
